JavaScript简介

JavaScript的构成

  1. ECMAScript：即ECMA-262定义的JavaScript标准化规范

    ES6新添加了一些特性，正式支持了类、模块、迭代器、生成器、箭头函数、期约、反射、代理和众多新的数据类型。

  2. 文档对象模型（DOM）：提供与网页内容交互的方法和接口

  3. 浏览器对象模型（BOM）：提供与浏览器交互的方法和接口

JavaScript的特点

  1. 解释型语言

  2. 类似于C、Java的语法结构

  3. 动态语言

    指程序在运行时可以改变其结构：例如新的函数、对象、甚至代码可以被引进，已有的函数可以被删除。

  4. 基于原型的面向对象

变量

ECMAScript变量是松散类型的，可以用var，let，const关键字定义变量。

作用域 可声明提升 可冗余声明
var 函数作用域 yes yes
let 块作用域 no no
const 块作用域 no no

  • 暂时性死区：在let声明变量之前，该变量是不可用的，此时引用未声明的let变量会抛出ReferenceError。

  • 用const声明变量时，必须同时初始化变量，且限制该变量的引用不可变（对象的属性还是可变的）。

  • 省略以上关键字，可定义一个全局变量。

作用域

作用域指一个变量的作用范围

1）全局作用域

  • 全局作用域在页面打开时创建，在页面关闭时销毁
  • 在全局作用域中有一个全局对象window。全局变量都会作为window对象的属性保存

2）函数作用域

3）块作用域

变量提升与函数提升

  • 变量提升：使用var声明的变量会自动提升到函数作用域顶部
  • 函数提升：只有函数声明能被提升到顶部，函数表达式不可以

垃圾回收

JavaScript有自动垃圾回收机制，基本思路为：确定哪个变量不会再使用，然后释放它占用的内存。当一个对象没有任何变量或属性对它进行引用，此时我们将永远无法操作该对象，此类对象需要进行清理。

数据类型

ECMAScript有6种简单数据类型（也称为原始类型）: Undefined、Null、Boolean、Number、String和Symbol

除了这6种简单数据类型外，还有一种引用数据类型叫Object。

因为ECMAScript的类型系统是松散的，可以用typeof操作符来确定任意变量的数据类型。

基本数据类型

  • Undefined：未定义
  • Null：空对象指针
  • Boolean：布尔
  • Number：数字
  • String：字符串
  • Symbol：符号

引用数据类型

1）分类

  • 对象（Object）

  • 数组（Array）

  • 函数（Function）

其实数组和函数实际上都是对象。

2）引用值

引用值是保存在内存中的对象。与其他语言不同，JavaScript不允许直接访问内存位置。在操作对象时，实际上操作的是对该对象的引用（reference）而非实际的对象本身。

  • JavaScript 的变量都是保存在栈内存中的，包括变量名和变量值
  • 对象是保存在堆内存中的，而变量保存的是对象的内存地址（对象的引用）
  • 若变量a和变量x指向同一个对象，我们可以更改a和x共同指向的值，但不能通过引用x来更改引用a的指向。

类型转换

1）将其他类型转换为 String

  • 可调用toString()方法，但null和undefined值没有toString()方法
  • 可使用String()转型函数，并将待转换的值作为其参数

2）将其他数据类型转换为Number

  • Number()，若字符串中有非数字内容，会转换为NaN
  • parseInt()
    • 专门针对字符串。忽略前面空格，直到遇到非字符为止
    • 对非String类型使用parseInt()会先转换为String再操作
    • 指定第二个参数为进制数，可转换进制（十六进制以0x开头，八进制以0开头，二进制以0b开头）
  • parseFloat()，转成浮点数

3）将其他数据类型转换为Boolean

  • 使用Boolean函数

对象

对象属于一种复合的数据类型，包含相关数据和方法。

ECMA-262将对象定义为一组属性的无序集合。对象中的内容就是一组名/值对，值可以是数据或者函数。

对象的分类

  • 内建对象：由ES标准中定义的对象

  • 自建对象

创建对象

使用Object构造函数或对象字面量创建对象的弊端：创建具有同样接口的多个对象需要重复编写很多代码。

工厂模式

使用工厂方法可以大批量创建类似对象。

工厂模式的缺点：不能区分新创建的对象的类型。因为使用工厂模式创建的对象，使用的构造函数都是Object，所以创建的对象都是Object这个类型。

构造函数

1）构造函数与普通函数唯一的区别就是调用方式不同。构造函数需要使用new操作符来调用。

2）构造函数的执行流程：

  1. 立刻创建一个新的对象
  2. 将新建的对象设置为函数中this，在构造函数中可以使用this来引用新建的对象
  3. 逐行执行函数中的代码（给新对象添加属性）
  4. 将新建的对象作为返回值返回

3）定义自定义构造函数可以确保实例被标识为特定类型。

使用同一个构造函数创建的对象，我们称为一类对象，也将一个构造函数称为一个类。我们将通过构造函数创建的对象，称为这个类的实例。

可以用instanceof操作符检查一个对象是否是一个类的实例。

4）构造函数的主要问题在于，其定义的方法会在每个实例上都创建一遍。

解决的办法：将函数定义移到构造函数外部，让所有对象共享同一个方法。

这样虽然解决了相同逻辑的函数重复定义的问题，但会污染全局作用域

原型模式

1）理解原型

每个函数都会创建一个prototype属性，这个属性指向原型对象。

当我们使用构造函数创建一个新的实例对象时，这个实例有一个内部[[Prototype]]属性，指向该构造函数的原型对象。[[Prototype]]是内部属性，不能被直接访问。但Firefox、Safari和Chrome浏览器为我们提供了一个__proto__属性，通过这个属性可以访问对象的原型（即实例的内部属性[[Prototype]]）。

原型的优点：原型对象就相当于一个公共的区域，在它上面定义的属性和方法可以被对象实例共享，这样就不用为每个实例都创建一个方法了。我们可以将对象中共有的内容，统一设置到原型对象中。

2）原型层级

每个对象都有一个原型对象，该原型对象也有自己的原型对象，层层向上直到找到Object对象的原型（Object对象的原型为null，如果在Object中依然没有找到，则返回undefined）。这样便形成了一条原型链。

当我们访问对象的一个属性或方法时，它会先在对象自身中寻找，如果有则直接使用，如果没有则会去原型对象中寻找。如此层层向上搜索，直到找到匹配的属性或达到原型链的末尾。

3）检查对象是否包含某个属性

  • 使用in操作符，如果对象中没有但是原型中有，返回true
  • 使用对象的hasOwnProperty()，检查对象自身是否包含该属性

函数参数

ECMAScript函数既不关心传入的参数个数，也不关心这些参数的数据类型。

因为ECMAScript函数的参数会被保存在一个类似数组的arguments对象中，但函数并不关心这个数组中包含什么。

因此，函数有可能接收到非法的参数，有些情况下需要对参数进行类型检查。如果函数实参个数大于形参个数，那么多出的实参不会被赋值给形参，但可以用arguments对象获取值；如果函数实参个数小于形参个数，未能够初始化的形参的默认值为undefined。

函数返回值

  • return语句可以不带返回值，用于提前终止函数执行

  • 返回值可以是任意数据类型，可以是对象，函数

函数作用域

  • 在函数内定义的变量不能在函数之外的任何地方访问，而在函数作用域中可以访问到全局作用域的变量
  • 当在函数作用域操作一个变量时，先在自身作用域寻找，找不到就向上一级作用域寻找
  • 在函数中要访问全局变量可以使用window对象，免得和函数作用域的变量冲突

函数内部对象

this

this引用的是函数调用的上下文对象。根据函数调用方式的不同，this会指向不同的对象：

  1. 在网页的全局上下文中调用函数时，this指向windows
  2. 以方法的形式调用时，this指向调用方法的那个对象
  3. 以构造函数调用时，this就是新创建的那个对象
  4. 使用call和apply调用时，this是指定的那个对象

arguments

arguments是一个类数组对象（但不是Array的实例），包含调用函数时传入的所有参数。可以使用中括号语法访问其中的元素。

立即执行函数

IIFE（ 立即调用函数表达式）是一个在定义时就会立即执行的函数。

()里包含的是一个匿名函数，表示一个函数表达式。然后再用一对括号()立即调用这个函数。

立即执行函数可以模拟块级作用域。

DOM

文档对象模型（DOM, Document Object Model）是HTML和XML文档的编程接口。DOM表示由多层节点构成的文档，通过它开发者可以添加、删除和修改页面的各个部分。

  • 文档（Document ）：整个 HTML 文档
  • 对象（Object）：将网页中的每一个部分都转换为了一个对象
  • 模型（Model）：将文档抽象表示为一棵具有层次结构的DOM节点树，更清晰地表示对象之间的关系，方便我们获取对象

事件

事件对象

当事件的响应函数被触发时，浏览器每次都会将一个事件对象作为实参传递进响应函数。在事件对象中封装了当前事件相关的一切信息，比如：鼠标的坐标，键盘哪个按键被按下，鼠标滚轮滚动的方向。

事件冒泡

所谓的冒泡就是事件的向上传导，当后代元素上的事件被触发时，其祖先元素的相同事件也会被触发。

在开发中大部分情况冒泡都是有用的，如果不希望发生事件冒泡可以通过事件对象来取消冒泡。

事件委派

指将事件统一绑定给元素的共同的祖先元素，这样当后代元素上的事件触发时，会一直冒泡到祖先元素。从而通过祖先的响应函数处理事件。

事件委派利用了事件冒泡，通过委派可以减少事件绑定的次数，提高程序的性能。

事件绑定

1）对象.事件 = 响应函数

只能为一个元素的一个事件绑定一个响应函数。不能绑定多个，否则会覆盖。

2）addEventListener()

使用addEventListener()可以为一个元素相同事件绑定多个响应函数，按照响应函数的绑定顺序执行。

参数：

  • 事件的字符串，不要’on’
  • 回调函数，当事件触发时该函数会被调用
  • 是否在捕获阶段触发事件，默认false

DOM事件流

DOM2 Events规范规定事件流分为3个阶段：事件捕获、到达目标和事件冒泡

1）捕获阶段

在捕获阶段时从最外层的祖先元素，向目标元素进行事件的捕获，但是默认不会触发事件

2）目标阶段

事件捕获到目标元素，捕获结束后开始在目标元素触发事件

3）冒泡阶段

事件从目标元素向它的祖先元素传递，依次触发祖先元素上的事件

事件类型

  • 用户界面事件
  • 焦点事件
  • 鼠标事件
  • 滚轮时间
  • 输入事件
  • 键盘事件
  • 合成事件

BOM

BOM（Browser Object Model），即浏览器对象模型。BOM提供了一组浏览器功能对象，用来完成对浏览器的操作。

window

代表整个浏览器的窗口，同时window也是网页中的全局对象。

navigator

代表当前浏览器的信息，通过该对象可以识别不同的浏览器

location

代表当前浏览器的地址栏信息。通过location对象可以获取地址栏信息，或者操作浏览器跳转页面。
